FAQs
Q: What is MythTV and what does this app do?
MythTV is an open-source DVR program (somewhat like TiVo) that runs on Linux.
If you do not already have MythTV setup this app is not for you.
If you do have a MythTV setup this app can be used a remote control and do some advanced features like showing your current and upcoming recordings.
Q: What is the script file noted in the preferences?
Earlier versions of the app required a server script to access all of the MythTV functions. The app is still compatable with the script and it can be used if desired by setting it in the preferences. Using the script is a good way to avoid having to forward all the ports when using the app from a cell network.
Q: The upcoming list/frontend searches/remote connection are not working, especially after I just upgraded the script file
First double check you installed the script file is installed on a local web server. And that you changed the hardcoded database values to your specific system. You can get the variables you need by looking at /etc/mythtv/mysql.txt on a functioning frontend or backend. You can see if it is accessible by trying to view it from a web browser (i.e http://my-server/cgi-bin/webmyth.py?op=getHosts). If your browser tries to download a file then that much is working. If you get a 500 error from your browser you can try looking at the apache error log ("/var/log/apache2/error.log") for information.
Q: WebMyth can't seem to talk to a remote frontend even though I know it is a valid frontend?
Make sure the IP address and port is correct. The app does not use the hostname to connect to the fronted, only the IP address.
Q: Can I use WebMyth remotely over the cell network or another WiFi network?
For the most part, yes. You will have to know the IP address assigned to you by your internet provider (or setup a domain name that resolves to your IP address). Then on your router you should forward ports 80 (webserver), 3306 (MySQL), 6543 (MythTV backend), 6544 (MythTV backend XML) and 6546 (frontend remote control) to the correct local IP address(es). You can also download a small addon module for MythWeb to avoid having to forward port 6544. See the app homepage for details. If these ports are blocked by your ISP or the network connection you are using you are out of luck.
Q: Why doesn't video playback work for all videos?
I haven't found a way to reliably play back video files from remote storage directories. Hopefully playback of video files that are not in sub-directories of your Videos storage group will work remotely, but if the video is in a sub-directory the app gets the frontend to try and play the file locally. For now I recommend mounting the remote Videos storage group locally via NFS for playback. (And I know having MythTV show video information during playback is not working.)
